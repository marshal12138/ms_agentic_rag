# 环境参数

本节整理 `tasks/train_tasks/coAgenticRetriever/train_CAR_async_ranker_training_ds_flash_mix_signal_fix.sh` 中第 6-26 行导出的环境变量，并说明这些变量在下游训练 launcher `scripts/coagenticRetriever_v2/01_train_launcher.sh` 中是否也存在默认值。

## 参数对照

| 变量 | 训练脚本中是否有默认 | 如果 task 不设会怎样 |
|---|---:|---|
| `EXP_NAME` | 有变量位，但默认空 | 会报错要求提供 `EXP_NAME` |
| `GROUP_NAME` | 有 | 默认 `coAgenticRetriever` |
| `AGENT_GPU_IDS` | 有 | 默认 overlay 的 `resources.agent_gpu_ids`，否则 `0,1,2,3` |
| `RANK_GPU_ID` | 有 | 默认 overlay 的 `resources.rank_gpu_id`，否则 `4` |
| `RECALL_GPU_ID` | 有 | 默认 overlay 的 `resources.recall_gpu_id`，否则 `5` |
| `LLM_JUDGE_GPU_IDS` | 有 | 默认 overlay 的 `resources.llm_judge_gpu_ids`，否则 `6,7` |
| `ENABLE_ASYNC_RANKER_TRAINING` | 有 | canonical 模式默认 `1`，legacy 模式默认 `0` |
| `AUTO_START_RECALL_SERVICE` | 有 | 默认 `1` |
| `AUTO_STOP_RECALL_SERVICE` | 有 | 默认 `1` |
| `RECALL_SERVICE_WAIT_SECONDS` | 有 | 默认 `240` |
| `AUTO_START_LLM_JUDGE` | 有 | 默认 `0`，和 task 里的 `1` 不同 |
| `AUTO_STOP_LLM_JUDGE` | 有 | 默认 `0`，和 task 里的 `1` 不同 |
| `LLM_JUDGE_PREFLIGHT` | 有 | 默认 `1` |
| `LLM_JUDGE_WAIT_SECONDS` | 有 | 默认 `600` |
| `WAIT_FOR_GPUS` | 不在训练脚本 | 只在 task 调用的 `wait_for_gpus.sh` helper 里使用 |
| `WAIT_FOR_GPU_RELEASE` | 不在训练脚本 | 只控制 task 启动前是否等待 GPU |
| `WAIT_FOR_GPU_INTERVAL_SECONDS` | 不在训练脚本 | helper 默认 `30` |
| `WAIT_FOR_GPU_LABEL` | 不在训练脚本 | helper 默认 `GPU wait` |

## 关键结论

- `EXP_NAME` 不能直接删除，除非从外部环境传入。训练脚本会用它构造唯一的 `RUN_NAME`；如果为空，会报错要求提供 `EXP_NAME`。
- `AUTO_START_LLM_JUDGE=1` 和 `AUTO_STOP_LLM_JUDGE=1` 是 task 层显式打开的行为。如果删掉，训练脚本默认是 `0`，LLM judge 服务不会自动启动或自动停止。
- `WAIT_FOR_*` 这组变量不是训练脚本参数，而是 task 在调用训练前先执行 GPU 等待逻辑时使用的参数。删掉后训练脚本仍可执行，但不会按该 task 的方式提前等待 GPU 释放。
- GPU 分配变量既可以由 task 环境变量指定，也可以从 overlay YAML 的 `resources` 段读取默认值；外部环境变量优先级高于 overlay 默认值。
